# @cre/web-ui — Usage (internal)

This document is **package-local** and intended to be read by humans and AI tools consuming `@cre/web-ui` from other repositories.

## What this package provides

- React components (Button, Input, Modal, Table, etc.)
- Layout/typography primitives (Stack, Inline, Grid, Text, Heading, etc.)
- A token-based theme system driven by CSS custom properties (`--cre-*`)

## Installation

### pnpm workspace (recommended)

In a pnpm monorepo, depend on this package via workspace protocol:

```json
{
  "dependencies": {
    "@cre/web-ui": "workspace:*"
  }
}
```

### External repo (manual)

```bash
pnpm add @cre/web-ui
```

- `react` and `react-dom` are peer dependencies (React 18+).

## Required setup: theme provider

All UI components are designed to consume CSS variables (`var(--cre-*)`). Those variables are applied by `CreThemeProvider`.

### Minimal setup

```tsx
import { CreThemeProvider, Button, Stack, Text } from '@cre/web-ui';

export function App() {
  return (
    <CreThemeProvider initialMode="light">
      <Stack gap="micro">
        <Text>Hello</Text>
        <Button>Click me</Button>
      </Stack>
    </CreThemeProvider>
  );
}
```

### Controlled mode (recommended for app-level state)

```tsx
import { CreThemeProvider, useCreTheme, Button } from '@cre/web-ui';

function ThemeToggleButton() {
  const { mode, setMode } = useCreTheme();
  return (
    <Button onClick={() => setMode(mode === 'light' ? 'dark' : 'light')}>
      Switch to {mode === 'light' ? 'dark' : 'light'}
    </Button>
  );
}

export function App({ mode, setMode }: { mode: 'light' | 'dark'; setMode: (m: 'light' | 'dark') => void }) {
  return (
    <CreThemeProvider mode={mode} onModeChange={setMode}>
      <ThemeToggleButton />
    </CreThemeProvider>
  );
}
```

### Provider `scope`

`CreThemeProvider` can apply variables in two ways:

- `scope="local"` (default)
  - Wraps children in a `<div data-cre-theme=... style={cssVars}>...`
  - Good for embedding a themed subtree into an existing page
- `scope="global"`
  - Applies CSS variables onto `document.documentElement`
  - Also sets/removes `data-cre-theme` on the root element
  - Good when this design system owns the entire page

Example:

```tsx
<CreThemeProvider initialMode="dark" scope="global">
  <App />
</CreThemeProvider>
```

## Token contract (how styling works)

### Invariants

- Components should not depend on hardcoded hex values.
- Components should style via CSS custom properties: `var(--cre-*)`.
- Theme changes are implemented by swapping the values of the same variables.

### CSS variables

The theme system exposes variables like:

- `--cre-color-bg`
- `--cre-color-surface`
- `--cre-color-text`
- `--cre-color-border`
- `--cre-accent-*`
- `--cre-space-*`
- `--cre-radius-*`

These are generated by:

- `coreTokensToCssVars()` (static vars)
- `themeTokensToCssVars(createThemeTokens(mode))` (mode-dependent vars)

## Recommended composition patterns

### Primitives first

- Use primitives for layout and typography.
- Use components for interaction patterns.

Examples:

- Layout: `Container` + `Stack` + `Inline` + `Grid`
- Surfaces/regions: `Surface`, `Card`, `ScrollArea`
- Typography: `Heading` + `Text`

### Spacing must be token-driven

Avoid hardcoded spacing (`margin: 12`, `padding: 20`). Prefer:

- `Stack gap="micro"`
- `Inline gap="nano"`
- Primitive props that map to `--cre-space-*`

### Colors must reference CSS vars

Inline styles should use `var(--cre-*)` so they respond to mode changes:

```tsx
<div style={{ background: 'var(--cre-color-surface)', color: 'var(--cre-color-text)' }} />
```

## Troubleshooting

### "useCreTheme must be used within a CreThemeProvider"

Cause: you rendered a component calling `useCreTheme()` outside of a provider.

Fix: ensure that subtree is wrapped in `CreThemeProvider`.

### Styles look un-themed / wrong colors

Cause: the provider is missing, or variables are not applied to the area you expect.

Fix:

- Wrap the UI with `CreThemeProvider`.
- If the UI is rendered in a portal or outside your provider subtree, consider `scope="global"`.
