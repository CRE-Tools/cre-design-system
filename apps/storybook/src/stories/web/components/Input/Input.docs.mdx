# Input

A text input field with optional leading and trailing icon slots. Always pair Input with a `Field` component to provide a label, description, and error state.

---

## When to use

- Single-line text entry — search queries, names, emails, filters.
- Always wrap in `Field` when presenting to end users (label + error state).
- For multi-option selection use `Select`; for boolean state use `Checkbox`.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | — | Controlled value |
| `onChange` | `(value: string) => void` | — | Called on every keystroke with the new string value |
| `placeholder` | `string` | — | Placeholder text |
| `leading` | `React.ReactNode` | — | Content rendered inside the input on the left |
| `trailing` | `React.ReactNode` | — | Content rendered inside the input on the right |
| `disabled` | `boolean` | `false` | Disables the input |
| `autoComplete` | `string` | — | Native autocomplete attribute |
| `id` | `string` | — | Links to a `<Field htmlFor>` label |
| `className` | `string` | — | Additional CSS class |

---

## With Field

```tsx
<Field label="Email" htmlFor="email-input" error={error}>
  <Input
    id="email-input"
    value={email}
    onChange={setEmail}
    placeholder="you@example.com"
  />
</Field>
```
