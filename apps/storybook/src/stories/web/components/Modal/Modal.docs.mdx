# Modal

A centered overlay dialog for focused, disruptive interactions. Modal pauses background activity and requires the user to act before continuing.

---

## When to use

- Confirmation dialogs ("Are you sure you want to delete this?").
- Quick forms that don't warrant a full page navigation.
- Focused tasks that need the user's full attention before returning to the underlying page.
- Do **not** use Modal for progressive disclosure or information that the user needs to reference while working — use `Drawer` instead.

---

## Example

Click the button to open the modal. Dismiss by clicking outside or pressing Escape.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | — | Controls visibility. When `false`, the modal is unmounted. |
| `title` | `React.ReactNode` | — | Header title. A string renders as `Heading level={4}` with an `id` for `aria-labelledby`. |
| `children` | `React.ReactNode` | — | Body content |
| `footer` | `React.ReactNode` | — | Footer slot — typically action buttons |
| `onClose` | `() => void` | — | Called when the overlay is clicked or Escape is pressed |
| `dismissible` | `boolean` | `true` | If false, clicking outside or pressing Escape does nothing |
| `className` | `string` | — | Applied to the modal panel |

---

## Accessibility

- Renders as `role="dialog"` with `aria-modal="true"`.
- When `title` is a string, `aria-labelledby` is set automatically.
- Escape key triggers `onClose` when `dismissible` is true.
- Focus management: content is rendered inside the dialog element; consider adding a `autoFocus` to the first focusable element inside the body for keyboard users.
