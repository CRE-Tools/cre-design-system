# Card

A structured content container with optional header and footer sections. Built on `Surface variant="raised"` with consistent padding and border radius.

---

## When to use

- Grouping a self-contained unit of information (metric, form section, entity details).
- Pairing with Table — wrap the table in `<Card padding="none">` to give it a surface and border.
- Stacking multiple related content blocks inside a page.
- Do **not** nest Card inside Card — use Surface variants for inner groupings.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `header` | `React.ReactNode` | — | Content rendered above the body, separated by a border |
| `footer` | `React.ReactNode` | — | Content rendered below the body, separated by a border |
| `children` | `React.ReactNode` | — | Main body content |
| `padding` | `CreSpace \| 'none'` | `'micro'` | Inner padding applied to header, body, and footer |
| `variant` | `CreSurfaceVariant` | `'raised'` | Surface elevation variant |
| `className` | `string` | — | Additional CSS class |
| `style` | `React.CSSProperties` | — | Inline styles |

---

## Table inside Card

The most common pattern is wrapping a Table in a Card with `padding="none"` so the table's borders align with the card edge:

```tsx
<Card padding="none" header={<Heading level={4}>Records</Heading>}>
  <Table columns={columns} rows={rows} />
</Card>
```
